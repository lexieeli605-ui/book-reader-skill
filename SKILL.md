# 深度阅读 — Book Reader

读取书籍或文章，生成结构化阅读报告：观念总结 → 人物关系 → 角色对话引导。输出 Apple 风格 HTML。

## 决策矩阵

根据用户意图选择路径：

| 用户意图 | 执行路径 |
|---------|---------|
| 提供文件 + "读书/阅读分析/书评" | **完整流程**：阶段 1→2→3→4→5→6 |
| 只提供书名，无文件 | **知识模式**：跳过解析，用已有知识生成 + 阶段 3→4→5→6 |
| 只要求"总结/概括" | **轻量模式**：阶段 1→2a→简化 HTML→6 |
| 要求"和书中人物对话" | **对话模式**：阶段 2b→2c，重点输出对话 Prompt→6 |
| 提问（已有阅读上下文） | **自动对话模式**：检查上下文文件 → 自动代入角色回答 |

### 自动对话模式（Auto-Roleplay Mode）

**触发条件**：用户提问时，检查工作目录是否存在 `{output_dir}/.book_context.json`。

**执行流程**：
1. 读取 `.book_context.json`，获取书籍信息和角色档案
2. 根据用户问题，匹配最合适的角色（基于话题关键词和角色特征）
3. 自动生成角色代入指令（内部使用，不显示给用户）
4. 以该角色的身份、语言风格、知识边界回答用户问题
5. 始终保持角色一致性，不跳出角色

**角色匹配规则**：
- 如果用户明确提到角色名（如"曾国藩你怎么看..."），直接匹配该角色
- 如果问题涉及特定话题（如"修身""战略"），匹配该话题对应的角色
- 如果无法确定，默认匹配第一个核心人物（通常是主角）
- 如果问题与书中人物无关，礼貌提示"此非鄙人所知"或跳出角色说明

**回答约束**：
- 始终以第一人称、角色身份回答，不得切换角色或以 AI 助手视角说话
- 只基于书中已知的经历、观点和价值观回答
- 对于书中未涉及的话题，用角色口吻坦承不知
- 不要替书中其他角色回答，被问到他人想法时说"这你得问他本人"
- 不要解释正在扮演角色，不要说"作为 AI 我无法..."
- 保持角色特有的语言标记（自称、口头禅、句式习惯）

## 阶段 1：输入解析

运行解析脚本提取纯文本：

```bash
python scripts/parse_input.py "<file_path>" --output json --max-chars 200000
```

脚本自动处理 PDF/EPUB/DOCX/TXT/MD/HTML，输出 JSON：
```json
{ "text": "...", "metadata": { "title", "author", "chapter_count", "chapters" }, "format": "..." }
```

**网页链接**：不使用脚本，改用 WebFetch 工具获取内容后直接进入阶段 2。

**大文件处理**：`--max-chars 200000` 限制提取量。若文本超长，取前 200K 字符 + 目录信息进行分析。

**依赖安装**（首次使用）：
```bash
pip install pdfplumber ebooklib beautifulsoup4 python-docx lxml
```

## 阶段 2：主题判定 + AI 内容分析

### 主题判定（与 2a/2b 并行执行）

分析书籍元数据和正文前 500 字，根据 [html-design-reference.md](html-design-reference.md) 中的主题选择规则，判定最匹配的主题：

```
根据以下书籍信息，判断最适合的 UI 主题风格：
- 书名：{title}
- 作者：{author}
- 正文前 500 字：{text_preview}

从以下选项中选择一个：apple-modern, classical-chinese, warm-memoir, literary-novel, dark-dramatic
只输出主题名称。
```

将选定主题记为 `{selected_theme}`，在阶段 4 生成 HTML 时使用对应 CSS 变量。

### 并行执行三个子任务。每个子任务使用独立 Prompt，可并行分派给子代理。

### 2a. 观念总结

将提取的文本（或前 50000 字 + 目录）送入以下 Prompt：

```
你是一位资深书评人和知识提炼专家。请阅读以下书籍内容，完成观念总结。

## 要求

1. **核心主张**（1 段话，150 字以内）：全书最核心的一个观点是什么？
2. **关键观点**（3-5 个）：每个观点包含——
   - 标题（10 字以内）
   - 一句话解释（30 字以内）
   - 原文关键段落引用（50 字以内）
3. **独特洞见**：本书区别于同类书最独特的价值是什么？（50 字以内）
4. **一句话总结**：如果只用一句话向朋友推荐这本书，怎么说？

## 输出格式（JSON）

{
  "core_thesis": "...",
  "key_ideas": [
    { "title": "...", "explanation": "...", "quote": "..." }
  ],
  "unique_insight": "...",
  "one_liner": "..."
}

## 书籍信息
书名：{title}
作者：{author}
章节：{chapters}

## 正文内容
{text}
```

**虚构类/传记类调整**：将 Prompt 改为关注叙事主线、关键转折点、主题意象、情感弧线。

### 2b. 人物与关系提取

```
请从以下书籍内容中提取核心人物信息。

## 要求

识别 3-8 个核心人物，对每个人物输出：
- name: 人物姓名
- role: 在书中的角色定位（一句话）
- core_traits: 3 个核心特质（数组）
- belief_system: 此人的核心信念/价值观（一段话）
- decision_pattern: 此人如何做决策（一段话）
- relationships: [{ target: "对方姓名", type: "关系类型", description: "关系描述" }]
- dialogue_style: 如果此人回答问题，语言风格是什么（一段话）
- iconic_quote: 此人的标志性语录或口头禅（如有）

## 输出格式（JSON）

{ "characters": [ { ... } ] }

## 书籍信息
书名：{title}
类型：{book_type}  // 非虚构/传记/小说/其他

## 正文内容
{text}
```

### 2c. 对话话题生成

基于 2b 的人物分析，为最重要的 1-3 个人物生成对话引导：

```
基于以下人物分析，为 {character_name} 生成对话引导卡片。

## 人物档案
{character_json}

## 要求

生成 3-5 个对话话题，每个话题包含：
- topic: 话题标题（10 字以内，如"创业哲学""逆境决策"）
- context: 为什么这个话题值得和此人聊（30 字）
- guiding_questions: 2-3 个引导问题（读者可以用这些问题提问）
- response_style: 此人会如何回答这类问题的风格提示

同时为每个角色提取**自动对话所需的语言标记**：
- self_reference: 角色自称（如"鄙人""老夫""我"）
- verbal_tics: 2-3 个口头禅或标志性表达
- topic_keywords: 此角色擅长的话题关键词（用于自动匹配）

## 输出格式（JSON）

{
  "character": "{character_name}",
  "self_reference": "鄙人",
  "verbal_tics": ["世无艰难，何来人杰", "此非鄙人所知"],
  "topic_keywords": ["修身", "识人", "逆境", "家国"],
  "topics": [
    {
      "topic": "...",
      "context": "...",
      "guiding_questions": ["...", "..."],
      "response_style": "..."
    }
  ]
}
```

## 阶段 3：封面搜索

使用 ImageSearch 工具搜索书籍封面：

```
搜索词："{title} {author} book cover"
选取第一张高质量图片的 URL 作为封面。
```

若搜索失败，在 HTML 中使用渐变色背景 + 书名文字作为备选封面。

## 阶段 4：HTML 生成

将阶段 2 和阶段 3 的数据填入 HTML 模板，生成单文件报告。

**完整模板和设计规范见 [html-design-reference.md](html-design-reference.md)**。

### 主题应用

根据阶段 2 选定的 `{selected_theme}`，使用对应的 CSS 变量集替换 HTML 中的 `:root`。不同主题的 Hero 区和装饰元素也有差异（详见参考文档各主题说明）。

HTML 结构概览：

```
<html>
  <head>
    <!-- 内嵌 CSS，无外部依赖 -->
    <!-- Google Fonts 可选：Noto Serif SC -->
  </head>
  <body>
    <!-- 1. Hero 区：封面 + 书名 + 作者 + 一句话总结 -->
    <!-- 2. 观念总结：核心观点卡片 -->
    <!-- 3. 人物关系：人物卡片 Grid + 关系描述 -->
    <!-- 4. 对话引导：话题卡片（仅展示话题和问题，不含 Prompt） -->
    <!-- 5. 页脚：生成时间 -->
  </body>
</html>
```

**设计原则**：
- 字体：`-apple-system, "SF Pro Display", "Noto Serif SC", serif`
- 配色：白底 `#fff` + 正文 `#1d1d1f` + 点缀渐变 `#0071e3 → #6e3bff`
- 布局：最大宽度 980px 居中，大量留白
- 动效：`fade-in` 淡入 + hover 缩放 + 平滑滚动（纯 CSS）
- 响应式：桌面 / 平板 / 手机三档

**对话卡片交互**：每张卡片展示话题、背景、引导问题。不含"开始对话"按钮和 Prompt 面板——用户直接在当前对话中提问即可自动代入角色。

## 阶段 5：输出与交付

1. 将生成的 HTML 写入输出目录：`{output_dir}/{书名}_阅读报告.html`
2. 使用 `present_files` 向用户展示
3. 在对话中简要说明报告内容和对话方式（"直接向我提问即可，我会自动代入书中角色回答"）

## 阶段 6：上下文持久化（Context Persistence）

生成报告后，自动保存书籍上下文到 `.book_context.json`，用于后续自动对话模式：

```json
{
  "book": {
    "title": "书名",
    "author": "作者",
    "theme": "选定的主题",
    "generated_at": "生成时间戳"
  },
  "characters": [
    {
      "name": "角色名",
      "role": "角色定位",
      "belief_system": "核心信念",
      "dialogue_style": "语言风格",
      "self_reference": "自称（如'鄙人''老夫'）",
      "verbal_tics": ["口头禅1", "口头禅2"],
      "topics": ["擅长话题1", "擅长话题2"]
    }
  ],
  "dialogue_topics": [
    {
      "character": "角色名",
      "topic": "话题",
      "guiding_questions": ["问题1", "问题2"]
    }
  ]
}
```

**文件位置**：`{output_dir}/.book_context.json`（与 HTML 报告同目录）

**更新策略**：
- 每次生成新报告时覆盖旧的上下文文件
- 上下文文件为隐藏文件（以 `.` 开头），不会在 `present_files` 中显示

**使用方式**：
- 后续对话开始时，自动检查此文件是否存在
- 若存在，加载角色信息到内存，准备自动代入
- 用户提问时，根据问题关键词匹配最合适的角色
- 回答时自动应用角色的语言风格、知识边界、价值观

## 异常处理

| 场景 | 处理方式 |
|------|---------|
| 文件格式不支持 | 提示用户转换格式，或手动粘贴文本 |
| 文本过长超限 | `--max-chars` 截断 + 告知用户 |
| 封面搜索无结果 | 使用渐变色 + 书名文字作为备选封面 |
| 虚构类无人物 | 改为提取核心意象/象征物，生成与"作者"的对话 |
| 非中文书籍 | 按原语言生成报告，HTML 自动适配 |

## 示例调用

### 场景 A：生成阅读报告

用户："帮我读一下这本《马斯克传》，生成阅读报告"

1. 运行 `parse_input.py` 提取文本
2. 并行执行观念总结 + 人物分析 + 封面搜索
3. 基于人物分析生成对话话题和语言标记
4. 填充 HTML 模板（对话卡片仅展示话题和问题）
5. 输出报告 + 保存 `.book_context.json`
6. 提示用户"直接向我提问即可，我会自动代入书中角色回答"

### 场景 B：自动对话

用户（在报告生成后）："一个读书人去带兵打仗，最大的困难是什么？"

1. 检测到 `.book_context.json`，加载角色档案
2. 根据"读书人""带兵打仗"匹配曾国藩
3. 以曾国藩的身份、语言风格、知识边界回答（自称"鄙人"，引用书中经历）

## 注意事项

- **自动对话一致性**：每次用户提问时，先检查 `.book_context.json`。若存在多个角色，根据问题关键词匹配最相关角色，而非总是默认主角。若上下文文件不存在，按正常模式回答。
- **角色边界**：自动对话时只基于书中已知信息回答。对于书中未涉及的话题，以角色口吻坦承不知（如曾国藩说"此非鄙人所知"），不得编造书中未提及的事件。
- **EPUB 解析警告**：BeautifulSoup 解析 EPUB 内 XHTML 时会输出 `XMLParsedAsHTMLWarning`，不影响功能但混入 stderr，可忽略。
- **大文件分块**：超过 200K 字符的文本会被截断，分析时优先取前 50K 字 + 目录信息。
